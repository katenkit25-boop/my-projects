# Мультиагентный контент-бот

Бот сам пишет посты в Telegram силами 5 ролей и публикует их **только после твоего одобрения**.

**Как работает:** тема → 🔍 маркетолог (веб-поиск) → ✍️ копирайтер → 🎨 дизайнер (картинка) → 📲 вёрстка под Telegram → ✅ редактор → пост приходит тебе на проверку → ты отвечаешь «да» → бот публикует в канал.

- Текст пишут агенты через **Claude Code** (на твоей подписке, API-ключ не нужен).
- Картинки рисует **OpenAI** (нужен ключ).
- Бот работает через **polling** — без веб-сервера и вебхуков.

---

## Что нужно заранее

1. **Сервер (VPS)** с уже установленным и авторизованным **Claude Code** (тот же, где живёт твой основной агент).
2. **Node.js 18+** на сервере. Проверить: `node -v`.
3. **Бот в Telegram** — токен от [@BotFather](https://t.me/BotFather).
4. **Канал в Telegram**, куда постить, и бот, добавленный в него **администратором** с правом «Публикация сообщений».
5. **Ключ OpenAI** — с [platform.openai.com](https://platform.openai.com/api-keys) (для картинок).

---

## Установка на сервере (по шагам)

Открой на сервере терминал (VS Code → Remote Explorer → твой сервер → Terminal) и выполняй по очереди.

### 1. Скачать проект

```bash
cd ~/projects            # если папки нет: mkdir -p ~/projects && cd ~/projects
git clone https://github.com/katenkit25-boop/my-projects.git
cd "my-projects/Мультиагент"
```

### 2. Установить зависимости

```bash
npm install
```

### 3. Заполнить настройки

```bash
cp .env.example .env
nano .env                # впиши значения, потом Ctrl+O, Enter, Ctrl+X
```

Что вписать в `.env`:

| Переменная | Где взять |
|---|---|
| `TELEGRAM_BOT_TOKEN` | от @BotFather |
| `TELEGRAM_CHANNEL_ID` | `@имя_канала` или числовой `-100…` |
| `TELEGRAM_ADMIN_ID` | твой ID — напиши [@userinfobot](https://t.me/userinfobot) |
| `OPENAI_API_KEY` | platform.openai.com/api-keys |
| `TEXT_MODEL` | модель для текста (можно оставить по умолчанию) |
| `IMAGE_MODEL` | модель для картинок (`gpt-image-1`) |
| `POST_TIME` | во сколько постить, напр. `10:00` |

### 4. Проверить запуск

```bash
npm start
```

Если видишь `🤖 Мультиагентный бот запущен` — работает. Напиши боту в Telegram `/post денежное мышление` и проверь весь путь.

### 5. Оставить работать 24/7

Чтобы бот не выключался при закрытии терминала — через **pm2**:

```bash
npm install -g pm2
pm2 start index.js --name multiagent-bot
pm2 save
pm2 startup        # выполни команду, которую он подскажет — для автозапуска после перезагрузки
```

Полезное: `pm2 logs multiagent-bot` (логи), `pm2 restart multiagent-bot` (перезапуск).

## Как пользоваться

- **По расписанию:** каждый день в `POST_TIME` бот берёт верхнюю тему из `content-plan.md` и присылает тебе пост на проверку.
- **Вручную:** `/post <тема>` — собрать пост на любую тему прямо сейчас.
- **Проверка:** на присланный пост отвечаешь `да` (опубликовать), `правки: сделай короче` (переделать) или `нет` (отклонить).

## Темы

Список тем — в `content-plan.md`. Добавляй строкой `- [ ] Тема`. Опубликованные бот сам помечает `- [x]`.

## Сменить модель

Открой `.env` и поменяй `TEXT_MODEL` (текст) или `IMAGE_MODEL` (картинки). Перезапусти бота: `pm2 restart multiagent-bot`.

---

## Если что-то не так

- **«В .env не заполнена переменная…»** — открой `.env` и впиши значение.
- **Бот не публикует в канал** — проверь, что он админ канала с правом «Публикация сообщений», и что `TELEGRAM_CHANNEL_ID` верный.
- **Нет картинок** — проверь `OPENAI_API_KEY` и баланс OpenAI. Без картинки пост всё равно выйдет текстом.
- **Claude не отвечает** — на сервере должен быть авторизован Claude Code (`claude` в терминале работает).
