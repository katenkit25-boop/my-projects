# План: автопостинг с проверкой (передача на сервер)

> Этот файл — «эстафетная палочка». Мы спроектировали цепочку на компьютере,
> дальше собираем её НА СЕРВЕРЕ (рядом с ботом). На сервере: прочитай этот файл и продолжай с раздела «Где остановились».

## Цель

Агент по расписанию сам пишет пост (через 5 ролей), присылает Екатерине на проверку в Telegram, и публикует в канал ТОЛЬКО после её «да».

## Решения (утверждены Екатериной)

- **Режим:** автоматически по расписанию. По умолчанию 1 пост/день в 10:00 (потом подстроим).
- **Проверка:** бот присылает превью в личку, Екатерина отвечает текстом: «да» / «правки: …» / «нет».
- **Публикация:** только после «да». Без подтверждения в канал НИЧЕГО не уходит — никогда (красная зона).
- **Основа:** существующий бот на VPS (расписание + чат уже умеет). Добавляем только шаг публикации.
- **Канала пока НЕТ** — создаём с нуля (Шаг 0).

## Схема

```
[content-plan.md: очередь тем]
        │  (расписание, напр. каждый день в 10:00)
        ▼
[5 ролей: Исследователь→Копирайтер→Дизайнер→ТГ-агент→Редактор]
        │  черновик, статус = "на проверке"
        ▼
[бот шлёт превью в личку: «Публикуем? да / правки: … / нет»]
        │
   да ──► [публикация в канал] статус="опубликовано"
 правки ─► [правит и шлёт снова]
   нет ──► [в архив, тема пропущена]
```

## Кубики (каждый — простой отдельный файл)

1. `content-plan.md` — очередь тем по порядку. Агент берёт верхнюю неиспользованную.
2. Расписание — когда генерировать (по умолчанию ежедневно 10:00).
3. Генератор — прогон через 5 ролей (правила в CLAUDE.md Мультиагента).
4. `drafts/` — черновики со статусами: на проверке → опубликовано / отклонено (архив).
5. Проверка — превью в чат бота, ответ текстом.
6. Публикация — отправка финального HTML (+ обложка) в канал через Telegram Bot API.

## Правила безопасности

- Без «да» пост в канал не уходит НИКОГДА.
- Ошибка Telegram при отправке → статус НЕ «опубликовано», сообщить Екатерине.
- Кончились темы → бот пишет «темы закончились, добавь», не молчит.

## Порядок внедрения

- **Шаг 0 (Екатерина, в Telegram):** создать канал; добавить бота админом с правом «Публикация сообщений»; сообщить `@имя_канала` (или ID).
- **Шаг 1 (агент):** скрипт публикации + тест: бот реально пишет в канал.
- **Шаг 2 (агент):** `content-plan.md` — накидать первые 5-7 тем.
- **Шаг 3 (агент):** прописать шаг проверки (превью → ответ) в правилах Мультиагента.
- **Шаг 4 (агент):** подключить расписание на сервере (ежедневно 10:00), собрать цепочку.
- **Шаг 5 (тест):** прогнать готовый пост про денежное мышление весь путь: превью → «да» → публикация. Потом включить расписание.

## Где остановились

- Дизайн утверждён. Папка Мультиагент и черновик поста созданы ЛОКАЛЬНО (на компьютере).
- Переезжаем на сервер. Нужно: развернуть эту настройку рядом с ботом (`/home/agent/`), затем идти по Шагам 0→5.
- Готовый пост для теста (про денежное мышление) — см. историю чата / можно перегенерировать по теме «денежное мышление и результат».
