# Ручная переавторизация Claude

Если бот перестал отвечать или отвечает мгновенно пустыми сообщениями — скорее всего у Агента истёк «пропуск» к Claude. Эта инструкция поможет обновить пропуск за 5 минут.

Обычно пропуск обновляется автоматически, без Вашего участия. Но раз в год-два может понадобиться ручное обновление — это нормальная процедура безопасности.

---

## Когда нужна эта инструкция

Симптомы «протухшего» пропуска:

- Бот перестал отвечать в Telegram
- Отвечает мгновенно пустыми или обрезанными сообщениями
- Утренние брифинги приходят слишком быстро, без содержания

---

## Что вообще происходит

Claude выдаёт Агенту цифровой «пропуск», который живёт около суток. Перед каждым запросом Агент сам обновляет его — это автоматически. Но иногда обновление ломается:

- после смены пароля в Anthropic
- после случайного запуска команды `claude setup-token` в терминале
- после других редких причин

Тогда нужен новый «вход» — этим и займёмся.

---

## Пошаговая инструкция

### Шаг 1. Откройте VS Code и подключитесь к серверу

Это тот же путь, что Вы использовали при установке Агента:

1. Откройте VS Code на компьютере
2. Слева — иконка Remote Explorer (монитор со значком). Если не видите — нажмите Ctrl+Shift+P, наберите «Remote Explorer» и выберите
3. В разделе Tunnels найдите название Вашего сервера и кликните по нему
4. VS Code подключится к серверу

### Шаг 2. Откройте терминал

Нажмите Ctrl и ` (обратная кавычка — кнопка слева от цифры 1, там же где буква Ё).

Внизу экрана откроется тёмное окно — это терминал.

### Шаг 3. Переключитесь на пользователя бота

В терминале наберите:

```
sudo -u agent -i
```

Промпт (строка с курсором) должен поменяться на `agent@...:~$` — обратите внимание: **$ вместо #**. Это значит, Вы теперь работаете «от имени» бота, а не администратора.

> Если у Вас Агент работает не от пользователя `agent`, а от другого имени — подставьте Ваше имя. Проверить можно командой `systemctl cat agent-bot | grep User`.

### Шаг 4. Запустите вход в Claude

В терминале:

```
claude /login
```

- Если спросит «trust this folder» (доверяете папке) — нажмите `1` и Enter. Это разовая проверка безопасности
- Появится меню метода входа — выберите **1. Claude Pro/Max subscription**

> Важно: НЕ выбирайте «API key» или «Create token» — это другие методы, для бота не подойдут.

### Шаг 5. Авторизуйтесь в браузере

В терминале появится длинная ссылка вида `console.anthropic.com/oauth/authorize?...`. Скопируйте её целиком и откройте в браузере на своём компьютере.

Авторизуйтесь под аккаунтом Anthropic с подпиской **Pro** или **Max**. После нажатия «Authorize» Вам покажут короткий код возврата.

### Шаг 6. Вставьте код обратно в терминал

Скопируйте код, вернитесь в терминал, вставьте и нажмите Enter. Должны увидеть:

```
Login successful
Welcome back, <Ваше имя>!
```

### Шаг 7. Выйдите и перезапустите бота

В терминале:

```
exit
sudo systemctl restart agent-bot
```

### Шаг 8. Проверьте

Напишите Агенту любое сообщение в Telegram. Он должен ответить. Готово.

## Если через сутки опять перестанет работать

Значит «хвосты» от старого входа мешают автоматическому обновлению. Под пользователем root (или через `sudo`) выполните три команды:

```
sudo sed -i '/^CLAUDE_CODE_OAUTH_TOKEN=/d' /home/agent/.agent/.env
sudo sed -i '/^CLAUDE_CODE_OAUTH_REFRESH_TOKEN=/d' /home/agent/.agent/.env
sudo sed -i '/^CLAUDE_CODE_OAUTH_SCOPES=/d' /home/agent/.agent/.env
```

И повторите шаги 3–8. После этого «хвостов» не останется и пропуск будет обновляться сам.

> Эти команды убирают устаревшие переменные окружения, которые в редких случаях перекрывают автоматическое обновление. Это безопасно — настоящий пропуск хранится в другом файле.

---

## Типичные ошибки

- **«claude: command not found»** — Claude Code не установлен у этого пользователя. Проверьте `which claude` под нужным пользователем. Если пусто — нужна полная установка
- **Ссылка не открылась автоматически** — это нормально, сервер без графики. Просто скопируйте URL текстом и откройте на своём ноутбуке
- **Браузер ругается на аккаунт** — убедитесь, что в браузере Вы залогинены именно под тем Anthropic-аккаунтом, на котором куплен Claude Pro/Max
- **«Permission denied» при restart agent-bot** — нужны права администратора. Сначала наберите `exit` чтобы выйти из пользователя `agent`, потом `sudo systemctl restart agent-bot`

---

## Почему это нужно понимать

Авторизация через подписку Claude — это не «технический баг», а нормальная процедура безопасности. Anthropic ограничивает срок жизни «пропусков», чтобы их нельзя было украсть и использовать вечно. Обычно обновление работает само, но раз в год-два-три может потребоваться ручной вход — это нормально.

После ручного входа в Claude следующие обновления пропуска снова пойдут автоматически. Эта процедура — разовая, не на каждый день.
